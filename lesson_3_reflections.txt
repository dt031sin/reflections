When would you want to use a remote repository rather than keeping all your work local?

Ans. When i need to access it from different locations or to share it with others.

Why might you want to always pull changes manually rather than having Git automatically stay up-to-date with your remote repository?

Ans. Manual pull gives more control and it does not make changes automatically when someone changes the remote repository.

Describe the differences between forks, clones, and branches. When would you use one instead of another?

Ans. Forks : Github (from an account) to github (to your own), Clone: any git repository to anywhere except fork, branches: new section of main code to work on a new feature, etc.

What is the benefit of having a copy of the last known state of the remote stored locally?

Ans. So we can keep track of all the local changes made since.

How would you collaborate without using Git or GitHub? What would be easier, and what would be harder?

Ans. Collaborating without github will make changes to the code by asking the others every time a need to make the change and every single time others need to update their changes too.

When would you want to make changes in a separate branch rather than directly in master? What benefits does each approach have?

Ans. When i need to test something out. Master: easy & simple, Separate branch: more control over what changes to apply or decline.
